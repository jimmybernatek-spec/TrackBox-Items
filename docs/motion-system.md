# TrackBox Motion System

## Goal
Create a premium cinematic motion system for the entire TrackBox website.

Motion should make the website feel alive, immersive, smooth, and athletic.

## Overall Motion Style
Motion should feel:
- Smooth
- Controlled
- Premium
- Athletic
- Cinematic
- Intentional
- Fast but refined

Avoid:
- Cheap bounce animations
- Excessive spinning
- Overly playful motion
- Random motion timing
- Abrupt transitions

## Main Motion Libraries
Preferred:
- GSAP
- GSAP ScrollTrigger
- Framer Motion
- Spline interactions

## Scroll Motion
Use scroll motion heavily.

Examples:
- Product reveals
- Text fades
- Image parallax
- Camera movement
- Layer depth movement
- Section transitions
- Orbit scenes
- Product stacking animations

## Section Transition Rules
Never hard cut between sections.

Use:
- Fade overlays
- Shared lighting
- Motion carryover
- Gradient continuation
- Floating particles
- Product overlap
- Blur transitions

## Product Motion
Products should:
- Float subtly
- Rotate slowly
- Respond to hover
- Have depth
- Cast soft shadows
- Feel weightless but controlled

## Text Motion
Text should:
- Fade upward
- Sequence in smoothly
- Use staggered timing
- Avoid dramatic fly-ins

Headlines should feel strong and cinematic.

## Button Motion
Buttons should:
- Glow subtly on hover
- Slightly lift
- Animate smoothly
- Maintain premium simplicity

Avoid excessive button effects.

## Homepage Motion Flow
Preferred sequence:
1. 3D intro scene
2. Camera movement
3. Box opening
4. Orbiting products
5. Product callouts
6. Transition into homepage
7. Continued subtle parallax throughout the site

## Motion Speed Rules
Use:
- Smooth easing
- Longer premium transitions
- Controlled acceleration
- Cinematic timing

Avoid:
- Snappy cheap transitions
- Extremely fast movement
- Sudden scaling

## Layering Rules
Motion should create depth.

Use:
- Foreground movement
- Midground movement
- Background drift
- Light movement
- Floating particles
- Blur depth

## Mobile Motion Rules
Mobile should:
- Simplify animations
- Maintain smoothness
- Reduce object counts
- Prioritize performance
- Keep the premium feeling

## Codex Instructions
Codex should treat motion as part of the TrackBox brand identity.

Do not build static generic sections.

Every major section should contain:
- subtle motion
- depth
- transition continuity
- cinematic timing
- responsive behavior
